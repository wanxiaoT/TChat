## 三、优先级最高的优化项

### 1. 调整 Gradle 构建参数

**位置**

- `gradle.properties`

**当前观察**

```properties
org.gradle.jvmargs=-Xmx2048m -Dfile.encoding=UTF-8
# org.gradle.parallel=true
org.gradle.vfs.watch=false
org.gradle.workers.max=1
```

对于当前这种多模块、Compose、KSP、NDK 项目，这套配置过于保守，会明显拖慢日常构建、测试和安装。

**建议调整**

建议逐项验证后改为：

```properties
org.gradle.jvmargs=-Xmx4096m -Dfile.encoding=UTF-8
org.gradle.parallel=true
org.gradle.vfs.watch=true
```

并移除或提高：

```properties
org.gradle.workers.max=1
```

如果该限制最初是为了规避 Windows 文件锁或 Gradle cache move 问题，建议不要一次性全开。可以按以下顺序验证：

1. 先把 JVM heap 提到 `4096m`。
2. 开启 `org.gradle.parallel=true`。
3. 移除 `org.gradle.workers.max=1` 或提高到 `4`。
4. 最后再恢复 `org.gradle.vfs.watch=true`。

每一步验证：

```powershell
.\gradlew.bat testDebugUnitTest
.\gradlew.bat lintDebug
.\gradlew.bat :app:assembleDebug
```

**位置**

**当前观察**

仍需优化：

- `release` 目前 `isMinifyEnabled = false`。
- `release` 目前 `isShrinkResources = false`。

**建议动作**

1. 发布优先使用 AAB，让商店按设备拆分 ABI 和资源。
2. OCR 拆成 flavor 或可选功能模块。
3. JungleHelper 拆成独立 flavor 或可选模块。
4. Mermaid 改为按需加载、压缩替代或功能开关。
5. 如果坚持“不混淆”，可评估：

```kotlin
release {
    isMinifyEnabled = true
    isShrinkResources = true
    proguardFiles(
        getDefaultProguardFile("proguard-android-optimize.txt"),
        "proguard-rules.pro"
    )
}
```

并在 `proguard-rules.pro` 中使用：

```proguard
-dontobfuscate
```

这样可以让 R8 做 shrink / optimize，但不改类名。该方案必须完整验证启动、Provider、Room、TTS、OCR、二维码、JungleHelper、导入导出等路径。

### 5. 修复 Embedding Provider 并发取消语义

**位置**

- `network/src/main/java/com/tchat/network/provider/OpenAIEmbeddingProvider.kt`
- `network/src/main/java/com/tchat/network/provider/GeminiEmbeddingProvider.kt`

**当前观察**

主聊天 Provider 已经改用 `activeCalls`，但 Embedding Provider 仍使用单个：

```kotlin
private var currentCall: Call? = null
```

这在知识库批量索引、并行 embedding、用户取消任务时可能出现：

- 后一个请求覆盖前一个请求引用
- `cancel()` 取消的不是预期请求
- batch 之间取消语义不清晰

**建议动作**

改成 request-scoped call 或 `activeCalls`：

```kotlin
private val activeCalls = Collections.synchronizedSet(mutableSetOf<Call>())
```

每个 batch 内部：

1. 创建本地 `val call = client.newCall(request)`。
2. 加入 `activeCalls`。
3. `invokeOnCancellation { call.cancel() }`。
4. finally 中移除。
5. `cancel()` 取消 `activeCalls` 快照。

同时补测试或至少补手动验证：

- 并行两组 embedding 不互相覆盖。
- 取消一组不会影响另一组。
- batch 中途取消会停止后续网络请求。

## 四、中优先级优化项

### 6. 统一 TTS 调用路径

**位置**

- `feature-chat/src/main/java/com/tchat/feature/chat/ChatScreen.kt`
- `feature-chat/src/main/java/com/tchat/feature/chat/GroupChatScreen.kt`
- `data/src/main/java/com/tchat/data/tts/TtsService.kt`

**当前状态**

普通聊天已经走 `TtsService`：

```kotlin
val ttsService = remember(context) { TtsService.getInstance(context) }
```

群聊仍直接创建 Android `TextToSpeech`：

```kotlin
var tts by remember { mutableStateOf<TextToSpeech?>(null) }
```

**影响**

- 设置页 TTS 引擎、语速、音调可能不一致。
- 豆包 TTS 与系统 TTS 行为可能分叉。
- 生命周期释放逻辑重复。
- 自动朗读和手动朗读策略不统一。

**建议动作**

群聊页面改为只依赖 `TtsService`：

- 页面不直接创建 `TextToSpeech`。
- 手动朗读调用 `TtsService.speak()`。
- 停止朗读调用 `TtsService.stop()`。
- TTS 设置由统一服务读取。

### 10. 日志与隐私进一步收紧

**已完成改善**

`NetworkLogger` 已经：

- debug-only
- body 截断到 `4096` 字符
- 流式响应使用 bounded capture

**残留风险**

普通 `Log.d` / `Log.i` / `Log.e` 仍可能记录：

- 搜索 query
- MCP server URL
- SSE event data
- TTS 状态
- 下载 URL
- 第三方服务错误 body

当前 release 没有开启 shrink，也不会自动剔除日志。

**建议动作**

封装统一日志入口，例如 `AppLogger`：

- debug 输出详细日志。
- release 默认只输出必要 error。
- 禁止打印完整用户输入、完整 URL、完整 SSE payload、请求体、响应体、token、key。
- 敏感字段统一脱敏。

### 1. 已完成项

- 调整 Gradle 构建参数：
  - `org.gradle.jvmargs` 从 `-Xmx2048m` 提升到 `-Xmx4096m`。
  - 开启 `org.gradle.parallel=true`。
  - 恢复 `org.gradle.vfs.watch=true`。
  - 将 `org.gradle.workers.max` 从 `1` 提高到 `4`，避免 Windows 上完全放开 worker 后放大文件锁风险。

- 依赖版本迁移到 version catalog：
  - `data` 模块 OkHttp / OkHttp SSE / Gson / JSON test 依赖已改用 `libs.versions.toml`。
  - `feature-chat` 模块 Markwon / JLatexMath 依赖已改用 `libs.versions.toml`。
  - `data` 与 `network` 的 OkHttp 已统一到 catalog 中的 `5.3.2`。

- Embedding Provider 取消语义修复：
  - `OpenAIEmbeddingProvider` 从单个 `currentCall` 改为 `activeCalls`。
  - `GeminiEmbeddingProvider` 从单个 `currentCall` 改为 `activeCalls`。
  - 每个请求独立注册 `invokeOnCancellation { call.cancel() }`。
  - `cancel()` 会取消当前活跃请求快照，避免并发 embedding 时只取消最后一个请求。

- 群聊 TTS 路径统一：
  - `feature-chat` 群聊页面不再直接创建 Android `TextToSpeech`。
  - 手动朗读改为复用 `TtsService.getInstance(context)` 和 `TtsService.speak()`。
  - TTS 引擎、语速、音调、豆包 TTS 配置由统一服务承接。

- Lint 高价值警告清理：
  - 修复 `DefaultLocale`：数字格式化改用 `Locale.ROOT`。
  - 修复 `ClickableViewAccessibility`：自定义触摸视图补 `performClick()`。
  - 修复部分 `ModifierParameter`：将 Compose `modifier` 放到第一个可选参数位置。
  - 修复 `DatabaseBackupManager` 中 `getLongVersionCode` 的 API 兼容写法，改用 `PackageInfoCompat.getLongVersionCode()`。

- 测试覆盖：
  - 新增 `network/src/test/java/com/tchat/network/provider/EmbeddingProviderCancelTest.kt`。
  - 覆盖 OpenAI / Gemini 两个 embedding provider 的并发取消行为。
  - `network` 模块 JVM 单测添加真实 `org.json` test dependency，避免 Android stub JSON 影响 provider 单测。
  - `network` 模块开启 `unitTests.isReturnDefaultValues = true`，避免 OkHttp 在普通 Android JVM 单测中访问未 mock 的 Android `Log` 时失败。
